# 外贸销售 Agent 分级推进路线

## 产品目标

把系统从“询盘资料审查工具”升级为“外贸销售日常工作台”。

核心原则：

1. 先解决销售每天一定会打开系统的理由。
2. 先做人工确认闭环，再做自动化。
3. 先用已有数据生成工作流，再接邮箱、平台和 CRM。
4. AI 不直接承诺价格、交期、付款条款，只负责整理、提示、生成草稿和创建任务。

## P0：销售每天可用的工作流

目标：让销售打开系统就知道今天该处理什么。

### P0-1 今日待办

基于询盘案件生成待办：

- 有资料但未 AI 审查
- AI 审查后有缺失项，需要追问客户
- 有高风险项，需要内部确认
- 已生成邮件草稿但案件还没标记为待客户确认
- 已生成内部任务单但还没创建 v2 任务
- 案件已待客户确认，需要跟进

验收：

- 工作台顶部显示今日待办数量
- 每条待办能跳到对应案件
- 待办显示优先级、原因和建议动作

### P0-2 案件漏斗摘要

按案件状态聚合：

- 草稿
- 审查中
- 待客户确认
- 可报价
- 已关闭

验收：

- 销售能看到当前询盘池分布
- 可快速切换处理重点

### P0-3 报价前动作闭环

一个案件必须能完成：

资料归档 → AI 审查 → 人工确认 → 邮件追问 → 内部任务 → 状态流转

当前实现已基本覆盖，后续只需要增强可用性和提醒。

## P1：报价和样品流程

目标：从资料审查推进到报价和样品跟进。

### P1-1 报价草稿生成

不自动定价，只生成报价准备包：

- 报价假设
- 产品/规格摘要
- MOQ
- 样品费字段
- 包装字段
- 贸易条款
- 交期假设
- 客户待确认项

### P1-2 产品库匹配

根据结构化字段匹配：

- 产品类型
- 材料
- 尺寸
- 应用场景
- 目标市场

输出推荐产品和证据链接。

### P1-3 样品流程

状态：

- 样品费待确认
- 设计稿待确认
- 打样中
- 拍照确认
- 已寄样
- 已签收
- 等待反馈

## P2：渠道与客户资产沉淀

目标：减少手动录入，让系统成为客户资产库。

### P2-1 邮箱/平台/聊天同步

- 邮箱自动拉取
- 平台询盘导入
- WhatsApp/微信记录归档

### P2-2 客户画像

- 客户类型
- 沟通偏好
- 历史询盘
- 历史报价
- 压价倾向
- 付款习惯
- 风险标签

### P2-3 相似案件复用

- 推荐历史相似询盘
- 复用邮件追问模板
- 复用内部任务结构

## P3：管理视角和自动化

目标：从个人工作台升级到团队销售管理。

- 销售漏斗报表
- 平均响应时间
- 高价值询盘处理率
- 报价转样品率
- 样品转订单率
- 团队任务分配
- 自动提醒和自动归档

## 当前本轮实现目标

优先实现 P0-1 和 P0-2：

1. 后端新增销售工作台摘要接口。
2. 前端在询盘资料审查页顶部展示今日待办和案件漏斗。
3. 每条待办可以切换到对应案件。

## 本轮追加完成：P1 报价准备与跟进闭环

从资深外贸销售的真实使用场景出发，系统已经从“资料审查”继续推进到“报价前行动闭环”：

1. 报价准备包
   - 保存产品/规格摘要、报价假设、MOQ、样品费、样品交期、大货交期、贸易条款、目的港、付款条款和包装要求。
   - 自动计算报价准备完整度，并在工作台待办中提示准备不足的案件。
   - 创建内部任务时把报价准备包一起下发给 v2 AgentTask。

2. 样品和交期流程入口
   - 在询盘案件中沉淀样品费、样品交期和大货交期。
   - 这些字段不由 AI 承诺最终结果，只作为销售、工程、采购确认前的准备信息。

3. 客户追问闭环
   - 英文追问邮件可编辑、复制、保存。
   - 点击“已追问客户，进入跟进”后，系统会把追问内容归档为案件资料，状态切到“待客户确认”。
   - 可设置下次跟进时间，工作台会把到期跟进提升为高优先级待办。

4. 工作台待办增强
   - 支持识别资料未审查、缺失字段、报价风险、内部任务未下发、报价准备包不完整、客户跟进到期。
   - 销售打开系统后能先处理真正阻塞成交推进的事项。
